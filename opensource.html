Graphtastic

  • Clojure
  • Neo4j

Neo4j is a NoSQL Graph Database designed for high peformance transactional graphs. It is orders of magnitude more performant than relational databases for highly connected datasets where the primary usecase is navigation and path selection between them. Although, social network is the first thing that springs to one's mind when thinking of Neo4j, there are several problems that are amenable to graph solutions. For example, transitive authorizations and permissions can be modelled using graphs.

Graphtastic tries to bring in the awesomeness of Neo4j with the awesomeness of clojure. The traversal framework in Neo4j makes clojure a natural choice for writing expanders, predicate filters and evaluators.

irumugam

  • RSpec
  • Rack

Irumugam - The two faced framework helps you setup a DSL for describing services contracts. Contracts are described using a Sinatra like DSL with expectations. You can point it to an end point and run it as a RSpec suite or you can run it as a server and provide mocks for your services.

PG App Name

  • Ruby
  • RSpec
  • PostgreSQL

While we were debugging an polyglot application, we were setting the application name of the connections from various stacks to see if there were any issues with our connections. My colleague, who is an ardent fan of the Java way, jocularly remarked that setting the app name in Rails should be as simple declaring a gem in the Gemfile to make the life of lazy Rails programmers easier. I found there there was no such gem, so I wrote a gem which when used with postgres sets the application name after opening the postgres connection. Take that Java devs.

haml-sprockets

  • Ruby
  • RSpec
  • Sprockets
  • HamlJS

Rails 3.1 uses sprockets for its asset pipeline. Rails 3.1 has shown considerable love for SCSS and SASS but has left HAML out. We at Dharana Software Innovations are die hard HAML fans. We beleive that it is the best templating language around. So we would like to have it as a part of our client side toolkit as well. The haml-js project does exactly that. It was written for Node.js but works well with the browser too. This project helps us build JST templates with hamljs.

quartz-jruby

  • JRuby
  • Quartz
  • Java

Quartz is an open source Java based scheduling framework. This project provides a JRuby wrapper over the cron API. The API is meant to be readable and really simple to create cron jobs in the java process space.

RTFHighlight.vim

  • vim

This plugin is used to syntax highlight pieces of code to RTF and it automatically copies it to the OSX clipboard. This will be very useful in putting code into keynote presentations.
